# Proposed Skill Improvements

> **Self-Evolution Protocol, channel B** (see `_core-protocols.md` → Protocol 3).
>
> When an agent discovers something that should change a **skill file itself** — a missing rule, a
> better pattern, an outdated reference — it does **not** edit the skill silently. It appends a
> proposal here and asks the user. The user reviews, and only approved proposals are applied.
>
> Project-specific facts do **not** go here — those go straight into the brain (`learnings`,
> `conventions_used`, `key_decisions`). This file is only for changes to the framework's published
> skill handbook.

## How to add a proposal

Append a new entry at the top of the log using this template. Keep it concrete enough that a
reviewer can approve and apply it without re-deriving your reasoning.

```markdown
### [PI-<n>] <short title>
- **Status:** proposed        <!-- proposed | approved | rejected | applied -->
- **Raised by:** <agent-name> on <YYYY-MM-DD>
- **Target skill:** .claude/skills/<skill>.md  (section: <where>)
- **What:** <the concrete change>
- **Why:** <what triggered this; why it generalizes beyond this one project>
- **Suggested wording:**
  > <draft text to drop into the skill>
```
